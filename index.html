PROJECT 01 · 캡스톤디자인
산불 확산 예측과 웜봇 동선 최적화
기간
2025.03 – 2025.10
역할
산불 확산 예측 모델 구현

4인
도구
QGIS · Python
QGIS셀룰러 오토마타IDW 보간Greedy Set Cover다익스트라
배경

기존 산불 연구는 확산 예측에 머물러 대응 전략이 빠져 있었습니다. 산불 신고 후 진화 헬기 도착까지 약 50분의 골든타임 동안 확산을 선제적으로 막을 방법이 필요했습니다.

접근 과정
  1. 국토지리정보원의 약 635km² 고해상도 지형 데이터로 태백산을 QGIS에 구현하고 경사도를 계산
  2. 기상청 관측소 두 곳의 풍속, 풍향, 습도를 IDW 보간으로 전체 지역에 확장하고 Min-Max 정규화
  3. 풍속, 경사도, 습도 가중치(0.4 / 0.3 / 0.2)와 sigmoid, tanh 함수로 확산 위험도를 계산
  4. 셀룰러 오토마타로 90m 픽셀 단위 50분 후 확산 지도를 예측(위험도와 풍향 가중치 7:3)
  5. Greedy Set Cover로 웜봇 허브 위치를, 다익스트라로 경사와 속도를 반영한 최적 경로를 설계

배경

발주량을 사람의 경험으로 판단하면 재고 과잉과 품절이 반복됩니다. 데이터로 발주량을 예측해 두 문제를 동시에 줄일 수 있는지 직접 검증했습니다.

접근 과정
  1. 17,000건 84개 컬럼 데이터에서 VMI 공정의 리드타임 구조를 먼저 파악하고 날짜 기준으로 재배열
  2. 변수 중요도 분석에서 D+3일 투입 예정 수량이 발주량과 가장 강한 연관을 보여 핵심 독립변수로 설계
  3. LSTM, Decision Tree, Random Forest, KNN, SVM, GBoost, XGBoost 7개를 비교 검증
  4. GBoost(R² 92.54%)와 Random Forest가 높았지만 튜닝 후 과적합이 확인되어 안정적인 XGBoost를 최종 선택

PROJECT 03 · 셀렉트스타

배경

RAG 검색의 정확도는 임베딩 학습 데이터의 품질이 좌우합니다. 교육 도메인 검색 성능을 높이려면 의미적 연관성이 확보된 고품질 학습과 평가 데이터가 필요했습니다.

접근 과정
  1. 교육청과 학교 자료에서 학습용과 평가용 데이터를 구분해 총 13,300건의 데이터셋을 구성
  2. 부서별 질문 빈도 상위 70%를 추린 뒤 Agglomerative Clustering으로 부서 대표 질문을 선별
  3. 질의에 답할 수 있는 Positive 문서와 헷갈리는 Hard Negative 문서를 라벨링하고 기준을 정의
  4. 자동 선별을 위한 LLM as a Judge 프롬프트를 구축하고 사람 검수와 결합
  5. 임베딩 라벨링 정의서 V2.2를 공동 작성하고 코로나19, 개인정보 검수 가이드를 추가
